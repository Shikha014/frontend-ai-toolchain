# AI Workflow Comparison

## Introduction

For this assignment, I implemented the same login page twice using two different prompting styles. The goal was to compare the quality of AI-generated code when using a vague prompt versus a detailed prompt.

## Round 1

In the first round, I used a simple prompt asking ChatGPT to create a basic HTML login form. The generated code contained only the essential elements such as email, password, and a login button. It worked correctly but had no styling, validation, responsiveness, or accessibility improvements.

## Round 2

In the second round, I used a detailed prompt with clear requirements. I asked for a professional login page with responsive design, required validation, placeholders, accessible labels, CSS styling, rounded corners, hover effects, and clean code.

The generated result was significantly better. The page looked modern, was easier to use, and required very few manual changes.

## Comparison

The first version was functional but very basic. The second version had improved design, validation, accessibility, and responsiveness. Because the prompt contained detailed instructions, the AI produced higher-quality code that required less editing.

## AI Mistake

The AI initially generated code that required small formatting adjustments before saving. After reviewing the output, I corrected it manually.

## Lessons Learned

This assignment showed that writing detailed prompts leads to better AI-generated code. Adding clear requirements, constraints, and expected behavior helps reduce errors and improves overall code quality.
